# x402 Payments Guide

## Overview

The payments plugin lets a Solana Agent Kit wallet satisfy [x402](https://github.com/coinbase/x402) paywalls without exposing the private key. The plugin builds an x402-compatible signer directly from the agent’s `KeypairWallet`, generates the required payment header, and retries the protected request with the correct signature.

## Prerequisites

- Instantiate your agent with `KeypairWallet`; the plugin derives an x402 signer from the in-memory keypair. Browser or custodial wallets that do not expose signing bytes are not yet supported.
- Point the agent at the same cluster (`solana` or `solana-devnet`) that the resource server expects. The plugin auto-detects devnet based on the RPC endpoint.

## Making a Paid Request

```typescript
import PaymentsPlugin, { makeX402PaymentRequest } from "@solana-agent-kit/plugin-payments";

const agent = new SolanaAgentKit(wallet, rpcUrl, config).use(PaymentsPlugin);

const result = await makeX402PaymentRequest(
  agent,
  "https://example-resource.dev",
  "/premium/weather",
);

console.log(result.status); // Final HTTP status
console.log(result.data); // Parsed JSON, text, or base64-encoded binary
console.log(result.paymentInfo); // Decoded X-PAYMENT-RESPONSE header
```

1. The plugin issues an initial GET.
2. If the server returns HTTP 402, the challenge payload is parsed with `PaymentRequirementsSchema`.
3. A Solana signer is assembled from the agent wallet via `@solana/signers` and `createPaymentHeader`.
4. The request is retried with an `X-PAYMENT` header; success responses include the decoded payment receipt in `paymentInfo`.

## Inspecting Payment Requirements

To inspect the paywall without paying, call `getX402PaymentInfo(agent, baseURL, endpointPath)`. It first attempts a `HEAD` request for an existing `x-payment-response` header, then falls back to a GET challenge. The return shape is:

```ts
{
  challenge: { requirement, version } | null; // Max price, network, recipient, etc.
  paymentResponse: { success, transaction, network, payer } | null;
}
```

Use this to display pricing details or prompt a human for confirmation before invoking the paid request.

## Sample & Troubleshooting

- `x402Sample/` still demonstrates a lower-level client using `x402` directly; it is a good sanity check for resource server setup.
- Common failures:
  - *“No Solana-compatible payment requirement found”* – the server only exposed EVM networks.
  - *“Payment was attempted but the server still returned HTTP 402”* – the transaction failed on-chain or the paywall rejected the header; inspect the resource server logs and the decoded payment response.
